# 多语言路由重构方案

## 当前问题
- 使用查询参数 `?lang=xx` 切换语言
- 所有语言共享同一个URL
- 搜索引擎难以索引不同语言版本
- 缺少明确的语言路径结构

## 目标路由结构

### 英文（默认，无前缀）
- `/` - 首页
- `/tarot` - 塔罗牌
- `/career` - 职业分析
- `/face` - 面相
- `/palm` - 手相
- `/dream` - 解梦
- `/fortune` - 求签
- `/daily` - 每日一读
- `/stories` - 故事
- `/insights` - 洞察

### 中文（/zh 前缀）
- `/zh` - 首页
- `/zh/tarot` - 塔罗牌
- `/zh/career` - 职业分析
- 等等...

### 法语（/fr 前缀）
- `/fr` - 首页
- `/fr/tarot` - 塔罗牌
- 等等...

### 日语（/ja 前缀）
- `/ja` - 首页
- `/ja/tarot` - 塔罗牌
- 等等...

## Next.js 实现方案

### 方案选择：使用 `[lang]` 可选路由段

Next.js 15 App Router 支持可选路由段，我们可以使用 `[[...lang]]` 或创建平行的语言路由。

**推荐方案：** 创建 `[lang]` 动态路由 + 根路由（英文默认）

```
src/app/
├── (default)/          # 英文默认路由（无前缀）
│   ├── page.tsx
│   ├── tarot/
│   ├── career/
│   └── ...
├── [lang]/             # 其他语言路由（zh/fr/ja）
│   ├── page.tsx
│   ├── tarot/
│   ├── career/
│   └── ...
└── layout.tsx
```

## 实施步骤

### 1. 创建语言中间件（推荐）
使用 Next.js middleware 来处理语言检测和重定向。

### 2. 重构路由结构
- 保持英文路由不变（向后兼容）
- 为 zh/fr/ja 创建 `[lang]` 动态路由
- 在 `[lang]` 路由中验证语言参数

### 3. 更新语言上下文
- 从路由参数读取语言而不是查询参数
- 更新 `useLang` hook

### 4. 更新导航和链接
- 所有内部链接需要包含语言前缀
- 语言切换器需要更新URL结构

### 5. 更新 sitemap.xml
- 为每个语言生成独立的URL
- 添加 hreflang 标签

### 6. SEO 优化
- 在 layout.tsx 中添加 hreflang 标签
- 更新 canonical URL
- 更新 OpenGraph 和 Twitter 卡片

## 兼容性考虑

### 向后兼容
- 保持 `?lang=xx` 查询参数支持（通过 middleware 重定向）
- 英文路由保持不变

### 重定向规则
```
/?lang=zh → /zh
/tarot?lang=zh → /zh/tarot
/tarot?lang=fr → /fr/tarot
```

## 风险评估

### 高风险
- 大量文件需要修改
- 可能影响现有用户的书签
- 需要仔细测试所有页面

### 缓解措施
- 分阶段实施
- 保持查询参数兼容性
- 添加自动重定向
- 充分测试

## 时间估算
- 路由重构：2-3小时
- 测试和调试：1-2小时
- SEO优化：1小时
- 总计：4-6小时

## 建议

考虑到改动规模较大，我建议：

1. **先实施 middleware 方案**（较简单）
   - 使用 Next.js middleware 拦截请求
   - 根据路径前缀设置语言
   - 保持现有路由结构不变
   - 只需修改语言检测逻辑

2. **或者采用渐进式迁移**
   - 先为新页面使用新路由
   - 旧页面保持查询参数
   - 逐步迁移
